## Tippani 1.4.0-beta.1 — PR file-detection fixes (beta)

A bug-fix beta on top of `1.4.0-beta.0`. No new features; this makes tippani actually find the markdown in a PR when you don't spell out every flag. Reported by Kay Unkroth after `tippani <PR> --org=… --project="Power BI"` returned "0 markdown files changed" on a PR that clearly had markdown.

> ⚠️ **Beta.** Installs under the `next` dist-tag; `npm i -g tippani` still pulls the previous stable.

### What was broken

Running without `--repo`:

```bash
tippani 920770 --org=https://dev.azure.com/powerbi --project="Power BI"
#   0 .md file(s) changed.
# No markdown files changed in this PR.   (exit 1)
```

…even though the PR did change markdown.

**Root cause (two paths, one symptom):**

1. With no `--repo`, `ADO_REPO` defaulted to the *project* name (`"Power BI"`), but the PR lives in repo `powerbi-specs`. The changed-files lookup queried a repo that didn't match and came back empty. The PR *title* still loaded because `getPullRequestById` is a global lookup that never uses the repo/project.
2. URL-encoded config values (`Power%20BI`) were used verbatim — the [#54](https://github.com/mavaali/tippani/issues/54) variant of the same failure.

### What's fixed

- **Repo auto-detected from the PR.** After the PR loads, tippani re-points at the authoritative repository (stable GUID) carried on the PR object, so every repo-scoped call targets the real repo. `--repo` is now optional, and a wrong or omitted repo name self-corrects. (Fixes #54.)
- **URL-encoded config decoded.** `Power%20BI` → `Power BI` instead of silently returning 0 files.
- **Graceful "no markdown" message.** When a PR changes no `.md` files, tippani now lists the non-markdown files it *did* find (counts per extension + sample paths) and explains it reviews markdown only — instead of a terse one-liner.
- **Auth docs.** README and prompts lead with `az login` (no PAT needed); PAT is demoted to an optional fallback, with a note that PAT creation is often blocked by tenant policy.

### Tests

New `src/config-util.test.mjs` (24 tests) covers config decoding, extension parsing, PR-repo derivation (including the exact no-`--repo` failure above), and the non-markdown summary. Full suite: **217 passing**.
