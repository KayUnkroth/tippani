## Tippani 1.6.0 — Review UX

A review-optimized UX layer on top of the 1.5.0 reliability work: surgical agent edits, a three-view spec toggle, a feedback triage filter, a PR picker, editor find/replace, and a host-routable browser opener — with a security-hardening pass over the new surface. From [#67](https://github.com/mavaali/tippani/pull/67). `npm i -g tippani` now pulls this build.

### Review UX

- **Surgical spec edits (`edit_spec`).** An agent stages a small, anchored change to a spec (a guarded line-range edit or a find/replace) without resending the whole document. A pure engine resolves all edits against one snapshot and applies them atomically: a failed `oldString` guard, an overlap, an out-of-range line, or a zero-match `find` fails the whole call and stages nothing. Review-only, never committed.
- **Three-view toggle (`set_view`).** Read a spec as the committed **Current** text, the **Diff** of the staged proposal, or the full **Proposed** text — switchable by the reader and drivable by the agent, seq-gated so a background stage never flips your view.
- **Feedback triage filter (`set_feedback_filter`).** A filter bar on the feedback page (thread state, reviewer, file, free-text search) the reviewer drives and the agent can set to focus the triage.
- **PR picker (`list_prs` + `/prs` + `--browse`).** List the PRs to review as openable tiles; the portal can boot PR-less in a browse mode. A review inbox rather than needing the PR id up front.
- **Find & Replace** in the editor, a **persistent Bordeaux focus highlight** tracking thread and section, a **host-routable opener (`TIPPANI_OPEN_CMD`)**, and a **height-capped comment-thread pane**.

### Security & robustness

- **`edit_spec` honors the editor lock.** Staging returns `409 { code: "locked" }` while you hold the file open in edit mode, and the auto-load is guarded on `!isDirty()`, so an agent edit never silently overwrites your unsaved buffer.
- **Shell-free host opener.** `TIPPANI_OPEN_CMD` passes the URL as a discrete argv element to a shell-less spawn, so URL content can never inject a command.
- **Nav stays on the resolved same-origin target**, and a deliberate `?edit=1` deep-link survives a fresh browser.

### Under the hood

New pure engines (`spec-edit`, `feedback-filter`, `pr-criteria`, `nav-guard`) with unit tests, plus two end-to-end review-UX smokes (`npm run smoke:review-ux`). Full suite green.
